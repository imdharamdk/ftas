# Render Secret Strategy (FTAS AI Ecosystem)

## Goals
- Never commit plaintext provider keys to Git.
- Encrypt API keys at rest in gateway storage.
- Keep runtime secrets managed by Render environment settings.

## Required Render Secrets
Configure on `ftas-ai-gateway` service:
- `JWT_SECRET` (auto-generated by Blueprint)
- `API_KEY_ENCRYPTION_SECRET` (set manually; long random value)

Recommended length:
- 32+ characters (high-entropy random string)

## How API keys are protected
When an admin adds a provider key:
1. Gateway masks key for UI display.
2. Gateway encrypts full key using AES-256-GCM (`API_KEY_ENCRYPTION_SECRET`).
3. Gateway stores:
   - encrypted payload
   - IV + auth tag
   - SHA-256 hash for integrity validation
4. Gateway never returns plaintext key in responses.

## Operational Best Practices
1. Use separate Render environments (staging/prod) with different secrets.
2. Rotate `API_KEY_ENCRYPTION_SECRET` with a migration window:
   - temporarily support dual decrypt keys
   - re-encrypt stored provider keys
   - remove old key after migration
3. Restrict admin endpoints via RBAC + network policies.
4. Enable audit logging for key create/delete/validate actions.

## Blueprint behavior
- `render.yaml` uses `sync: false` for `API_KEY_ENCRYPTION_SECRET`.
- During initial Blueprint creation, Render asks for this secret.
- Secret is stored in Render, not Git.
